01O que é um banco de grafos?

Em vez de tabelas e linhas, você armazena
nós e relacionamentos — como num quadro branco.

  • Nó — uma entidade (substantivo: Pessoa, Empresa)
  • Relacionamento — ligação (verbo) com direção e tipo
  • Propriedades — chave-valor em nós e arestas

O relacionamento é cidadão de 1ª classe: ele é armazenado, não recalculado com JOIN a cada consulta.

02Pra que serve?

Quando a pergunta é sobre o caminho entre as coisas, não sobre as coisas isoladas.

  • Redes sociais — "amigos de amigos"
  • Recomendação — Netflix, Amazon
  • Fraude — anéis de transações
  • Rotas & logística — caminho mais curto
  • Busca / IA — Google PageRank
  • Investigação — "siga o dinheiro" 💸

Não é bala de prata: relatório, soma e média → o relacional ganha.

analogia do metrô
Relacional: "quantas estações tem a linha?" → conta Grafo: "qual a rota de A até B?" → o caminho

03Relacional vs. Grafo

mesma pergunta dos dois lados
"Em que empresa a Ana trabalha?"
RELACIONAL — 3 tabelas + 2 JOINs 
SELECT e.nome
  FROM pessoas p
  JOIN trabalha_em t ON t.pessoa_id = p.id
  JOIN empresas e   ON e.id = t.empresa_id
 WHERE p.nome = 'Ana';
GRAFO — o padrão é o desenho 
MATCH (p:Pessoa {nome:'Ana'})
      -[:TRABALHA_EM]->
      (e:Empresa)
RETURN e.nome;

Cada "pulo" a mais: no SQL é mais um JOIN; no grafo, só mais uma seta.

04"Olá, mundo" em Cypher

A "SQL dos grafos": declarativa e ASCII-art — () é nó, -[]-> é relação. CREATE grava, MATCH…RETURN consulta.

CypherSQL equivalente
CREATEINSERT
MATCH … RETURNSELECT … FROM

Padrão ISO desde 2024 — o GQL, mesmo comitê do SQL.

// 1 · criar
CREATE (ana:Pessoa  {nome:'Ana'}),
       (acme:Empresa {nome:'ACME'}),
       (ana)-[:TRABALHA_EM]->(acme);

// 2 · consultar (o código = o diagrama)
MATCH (p:Pessoa)-[:TRABALHA_EM]->(e:Empresa)
RETURN p.nome, e.nome;

06Demo — o pulo do gato (multi-hop)

CYPHER · 2 hops, "mesmo intermediário" 
MATCH (o:Officer)-[:officer_of]->(e1)
      <-[:intermediary_of]-(i)
      -[:intermediary_of]->(e2)
      <-[:officer_of]-(outro:Officer)
RETURN DISTINCT outro.name;
▶ ao vivo
  • Buscar "Mossack Fonseca" / "Brazil"
  • Clicar → vizinhança · expandir os pulos
O MESMO EM SQL · 4 JOINs 
SELECT DISTINCT outro.name
  FROM relationships r1
  JOIN relationships r2 ON r2.end_id=r1.end_id
  JOIN relationships r3 ON r3.start_id=r2.start_id
  JOIN relationships r4 ON r4.end_id=r3.end_id
  JOIN officers outro ON outro.node_id=r4.start_id;

Um lado é um desenho; o outro, um quebra-cabeça. Medido: Cypher 193×.

07SQL vs. Cypher — medido

Cypher (Neo4j) SQL (PostgreSQL) escala log — cada degrau = 10×

Na busca: empate. Depois a distância verde↔vermelho — a vantagem do grafo — cresce da esquerda pra direita.

08Por quê? Index-free adjacency

Cada nó guarda ponteiros diretos pros vizinhos. Seguir uma aresta é O(1) local — independe do tamanho do banco.

  • Grafo: o pulo custa o mesmo com 2 mil ou 2 bilhões de nós
  • SQL: cada hop é um JOIN que varre a tabela de novo

Por isso a vantagem do grafo cresce com a profundidade.

analogia da rede social Grafo = abrir o perfil de alguém no Instagram: a lista de quem ele segue já está ali, na mão. Um pulo, instantâneo.

JOIN do SQL = a cada amigo, abrir uma planilha com todas as amizades da plataforma inteira e filtrar.

Com bilhões de linhas, filtrar uma vez já dói. "Amigo do amigo do amigo" trava.

09Algoritmos de grafo (GDS)

  • PageRank — o algoritmo do Google: os nós mais influentes
  • Louvain — acha comunidades: grupos muito ligados entre si (ex.: uma família + suas empresas + o escritório em comum)
  • Caminhos / centralidade — nativos
▶ ao vivo (numa pessoa comum!)
  • Busca → clicar num nó → aba Algoritmos
  • PageRank → ranking · Louvain → repinta o grafo por cor de comunidade
CALL gds.pageRank.stream('sub')
YIELD nodeId, score
RETURN gds.util.asNode(nodeId).name,
       score
ORDER BY score DESC;

Em SQL puro:
não há equivalente.

Conclusão

Escrever nos dois é fácil.
Rodar, é outra história.

Do empate na busca rasa às centenas de vezes nos multi-hop — até onde o SQL não termina nem consegue expressar. Quanto mais a pergunta é sobre conexões, mais o grafo ganha.

Grafo não substitui o relacional — usamos os dois. Neo4j 5.26 · GDS · APOC · PostgreSQL 17 · Next.js · Docker.

Apêndice · trade-offs

Quando usar grafo (e quando não)

grafo brilha
  • Relacionamentos profundos e variáveis
  • Caminhos, padrões de conexão
  • Recomendação, fraude, knowledge graph
relacional ganha
  • Dados tabulares, agregações em colunas
  • Relatórios, BI, somas e médias
  • Transações simples e bem normalizadas

Persistência poliglota — cada banco no que faz melhor. Foi o que fizemos.
